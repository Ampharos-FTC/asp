Tuning

ASP Currently comes with only Mecanum Drive Train for the GoBildaPinpointComputer localization.
We do plan on adding other Drive Trains and Localization methods in the future.

Tuning | Drive Train

Each Drive Train has its own Tuning Settings.

Every Tuning has base PID variables, kI, kP and kD, for Translation and Heading.
And distance Tolerance (Inches and Degrees).

Tuning | Mecanum Drive Train

Here is an example for the Mecanum Tuning class : public class MyMecanumTuning extends ASPMecanumTuning { // Translation PID double kPT = 0.02; double kIT = 0.06; double kDT = 0.44; // Heading PID double kPH = 0.05; double kIH = 0.12; double kDH = 0.44; // Tolerance double translationTolerance = 2.3; // Inches double headinTolerance = 2; // Degrees // Mecanum : // Motors // Front Left public String frontLeftMotorName = ""; public DcMotor.ZeroPowerBehavior frontLeftMotorZPB = DcMotor.ZeroPowerBehavior.BRAKE; public DcMotorSimple.Direction frontLeftMotorDirection = DcMotorSimple.Direction.FORWARD; // Front Right public String frontRightMotorName = ""; public DcMotor.ZeroPowerBehavior frontRightMotorZPB = DcMotor.ZeroPowerBehavior.BRAKE; public DcMotorSimple.Direction frontRightMotorDirection = DcMotorSimple.Direction.FORWARD; // Back Left public String backLeftMotorName = ""; public DcMotor.ZeroPowerBehavior backLeftMotorZPB = DcMotor.ZeroPowerBehavior.BRAKE; public DcMotorSimple.Direction backLeftMotorDirection = DcMotorSimple.Direction.FORWARD; // Back Right public String backRightMotorName = ""; public DcMotor.ZeroPowerBehavior backRightMotorZPB = DcMotor.ZeroPowerBehavior.BRAKE; public DcMotorSimple.Direction backRightMotorDirection = DcMotorSimple.Direction.FORWARD; } Motor Settings go by what you have set for your robot.

Autonomous

Making an Autonomous with ASP should be really straight forward.
After you've tuned your robot, you can use ASP Visualizer to create Poses and Paths.

Poses

A Pose is the robot position inside of the field, it has X, Y and Heading, for simplicity X and Y are measured in Inches and Heading in Degrees.

Every ASP Pose2D needs 3 values, Robot X, Robot Y, and Robot Heading. ASPPose2D myPose1 = new Pose2D(67, 41, 5); ASPPose2D myPose2 = new Pose2D(67, 41, 25);

Paths

Paths are used by our Drive Trains
A Path in ASP consits of many Poses.

Making a new Path is very easy. ASPPose2D myPose1 = new Pose2D(67, 41, 5); new ASPPath().add(new ASPPose2D(0, 0, 0)) .add(new ASPPose2D(0, 15, 0)) .add(new ASPPose2D(0, 15, 25)) .add(myPose1);

Example

Here is an Example MecanumDriveTrain Autonomous class. 

        @Autonomous
        public class MyASPAutonomous extends LinearOpMode {
            ASPLocalizer localizer;
            ASPMecanumDriveTrain drive;

            @Override
            public void runOpMode() {
                // Initializes our Localizer
                localizer = new ASPPinpointLocalizer(
                        hardwareMap,
                        new ASPPose2D(0, 0, 0)
                );

                // Initializes our DriveTrain
                drive = new ASPMecanumDriveTrain(hardwareMap, localizer, new MyASPMecanumTuning());

                // Creates a Path
                drive.paths = new ASPPath().add(new ASPPose2D(0, 0, 0))
                .add(new ASPPose2D(0, 15, 0))
                .add(new ASPPose2D(0, 15, 25))
                .add(new ASPPose2D(0, 0, 0));

                waitForStart();

                while (opModeIsActive()) {
                    drive.update(); // Updates DriveTrain
                    drive.followTarget(); // Follows the path
                }
            }
        }
